Security & Compliance

Security

CloudWatch Application Signals implements multiple layers of security to protect your observability data and ensure compliance with industry standards.

Data Encryption

Encryption at Rest:

  • All trace data is encrypted using AES-256 encryption
  • RUM data is encrypted with AWS Key Management Service (KMS)
  • Synthetic monitoring data is stored encrypted
  • You can use customer-managed keys for additional control

Encryption in Transit:

  • All data transmission uses TLS 1.2 or higher
  • API calls to CloudWatch services are encrypted
  • RUM data collection uses HTTPS
  • Agent-to-collector communication is secured

Identity and Access Management

IAM Permissions:

Use the principle of least privilege when granting access to Application Signals:

Data Privacy and PII Protection

Data Sanitization:

  • Implement data masking for sensitive information
  • Use tokenization for PII in trace attributes
  • Configure data retention policies
  • Regular audit of collected data

Best Practices:

  • Never include passwords, API keys, or tokens in traces
  • Hash or mask email addresses and phone numbers
  • Remove query parameters from URLs that contain sensitive data
  • Use generic identifiers instead of personal information

Network Security

VPC Configuration:

  • Deploy agents within your VPC for enhanced security
  • Use VPC endpoints for CloudWatch API calls
  • Configure security groups to restrict traffic
  • Implement network ACLs for additional control

Agent Security:

  • Run agents with minimal required permissions
  • Keep agents updated with security patches
  • Monitor agent logs for security events
  • Use read-only access where possible

Data Privacy

Understanding how Application Signals handles your data privacy requirements.

Data Collection and Usage

What We Collect:

  • Application performance metrics and traces
  • Service dependency information
  • Error information and stack traces
  • System and infrastructure metadata

What We Don't Collect:

  • Application business data or user content
  • Database contents or file system data
  • User credentials or authentication tokens
  • Personally identifiable information (unless explicitly included)

Data Retention

Default Retention Periods:

  • Trace data: 30 days (configurable up to 1 year)
  • Metrics: 15 months (standard CloudWatch retention)
  • RUM data: 30 days (configurable up to 1 year)
  • Synthetics results: 90 days (configurable)

Data Deletion:

  • Use CloudWatch data retention policies
  • Implement automated data lifecycle management
  • Request data deletion through AWS Support if needed

Compliance Frameworks

Supported Standards:

  • SOC 2: Security, availability, and confidentiality controls
  • PCI DSS: Payment card industry data security standards
  • HIPAA: Health Insurance Portability and Accountability Act
  • GDPR: General Data Protection Regulation
  • FedRAMP: Federal Risk and Authorization Management Program

Compliance Guidance

Specific guidance for common compliance scenarios.

HIPAA Compliance

Requirements:

  • Encrypt protected health information (PHI) in traces
  • Implement access controls and audit logging
  • Use Business Associate Agreements (BAA) with AWS
  • Configure appropriate data retention policies

Implementation:

  • Mask PHI in trace attributes and span names
  • Use customer-managed KMS keys
  • Enable CloudTrail for audit logging
  • Configure VPC endpoints for data isolation

PCI DSS Compliance

Key Considerations:

  • Never include cardholder data in traces or logs
  • Implement network segmentation
  • Use encryption for data in transit and at rest
  • Regular security assessments and penetration testing

Best Practices:

  • Tokenize sensitive data before instrumentation
  • Use private subnets for Application Signals agents
  • Implement least-privilege IAM policies
  • Regular review of collected data

GDPR Compliance

Data Subject Rights:

  • Right to access personal data
  • Right to rectification
  • Right to erasure ("right to be forgotten")
  • Right to data portability

Implementation:

  • Minimize PII collection in telemetry data
  • Implement data anonymization techniques
  • Configure data retention based on business needs
  • Document data processing activities

Security Monitoring

Monitor your Application Signals deployment for security events.

Security Best Practices

  • Enable CloudTrail for API activity logging
  • Monitor IAM policy changes and access patterns
  • Set up alerts for unusual data access patterns
  • Regular security assessments and vulnerability scans
  • Implement multi-factor authentication (MFA)

Incident Response

  • Have an incident response plan for security events
  • Know how to disable Application Signals if compromised
  • Document escalation procedures
  • Regular incident response testing
