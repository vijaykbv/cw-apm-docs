Implementation Guides

Instrumentation

Instrumentation is the process of adding observability capabilities to your applications. CloudWatch Application Signals supports multiple instrumentation approaches to fit different use cases and preferences.

Auto-Instrumentation

Auto-instrumentation requires no code changes and is the recommended approach for getting started quickly.

Supported Runtimes:

  • Java: JVM-based applications (Spring Boot, Tomcat, etc.)
  • Python: Django, Flask, and other Python web frameworks
  • .NET: ASP.NET Core applications
  • Node.js: Express.js and other Node.js frameworks

How It Works:

The Application Signals agent automatically instruments common libraries and frameworks, capturing traces, metrics, and logs without requiring any application code changes.

Manual Instrumentation with OpenTelemetry

For fine-grained control and custom instrumentation, use the OpenTelemetry SDKs directly in your code.

Benefits:

  • Complete control over what gets instrumented
  • Custom spans and attributes
  • Vendor-neutral approach
  • Works across different observability platforms

AWS Distro for OpenTelemetry (ADOT)

AWS-optimized distribution of OpenTelemetry that provides seamless integration with CloudWatch services.

Features:

  • Pre-configured for AWS services
  • Automatic AWS metadata collection
  • Optimized for CloudWatch Application Signals
  • Support for Lambda, ECS, EKS, and EC2

Language-Specific Guides

Detailed implementation guides for each supported programming language.

Java Applications

Auto-Instrumentation:

java -javaagent:/path/to/aws-opentelemetry-agent.jar -jar your-app.jar

Manual Instrumentation:

import io.opentelemetry.api.trace.Tracer;
import io.opentelemetry.api.trace.Span;

// Create custom spans
Span span = tracer.spanBuilder("custom-operation").startSpan();
try {
    // Your code here
} finally {
    span.end();
}

Python Applications

Auto-Instrumentation:

opentelemetry-distro opentelemetry-instrumentation python your_app.py

Manual Instrumentation:

from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor

tracer = trace.get_tracer(__name__)

with tracer.start_as_span("custom-operation") as span:
    # Your code here
    span.set_attribute("custom.attribute", "value")

.NET Applications

Auto-Instrumentation:

dotnet run --additional-deps /path/to/aws-opentelemetry-agent

Manual Instrumentation:

using OpenTelemetry.Trace;

using var tracerProvider = Sdk.CreateTracerProviderBuilder()
    .AddSource("MyCompany.MyProduct.MyLibrary")
    .AddConsoleExporter()
    .Build();

var tracer = tracerProvider.GetTracer("MyCompany.MyProduct.MyLibrary");

using var span = tracer.StartActiveSpan("custom-operation");
{
    // Your code here
    span.SetAttribute("custom.attribute", "value");
}

Node.js Applications

Auto-Instrumentation:

node --require @aws/opentelemetry-auto-instrumentation your_app.js

Manual Instrumentation:

const { trace } = require('@opentelemetry/api');

const tracer = trace.getTracer('my-service');

const span = tracer.startSpan('custom-operation');
span.setAttribute('custom.attribute', 'value');
// Your code here
span.end();

Service Integration

Integrate Application Signals with your existing AWS services and infrastructure.

Amazon ECS

Enable Application Signals for containerized applications running on Amazon ECS.

Prerequisites:

  • ECS cluster with EC2 or Fargate launch type
  • Task execution role with appropriate permissions
  • Application running on supported runtime

Enable Application Signals:

  1. Update your task definition to include the Application Signals sidecar
  2. Add the required environment variables
  3. Deploy the updated task definition

Amazon EKS

Deploy Application Signals on Kubernetes clusters using the AWS Distro for OpenTelemetry Operator.

Installation Steps:

  1. Install the ADOT Operator using Helm
  2. Create an Instrumentation resource
  3. Annotate your deployments to enable auto-instrumentation
  4. Verify data collection in CloudWatch

AWS Lambda

Monitor serverless functions with Application Signals.

Enable Application Signals:

  1. Add the AWS managed Lambda layer for OpenTelemetry
  2. Configure environment variables for tracing
  3. Update function permissions if needed

Lambda functions automatically get traces and metrics without code changes when using supported runtimes.

Amazon EC2

Monitor applications running directly on EC2 instances.

Deployment Options:

  • Auto-Instrumentation: Install the Application Signals agent on your instances
  • Sidecar Pattern: Run the collector as a sidecar container
  • Manual Instrumentation: Use OpenTelemetry SDKs in your application code
